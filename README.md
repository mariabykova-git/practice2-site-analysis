# practice2-site-analysis
Работа по практике на тему увеличения эффективности сайта с помощью модели, которая предсказывает вероятность того, что пользователь совершит целевое действие (оставит заявку, закажет звонок и пр.) на сайте
# Анализ поведения пользователей сайта СберАвтоподписка

Учебный проект по анализу пользовательских сессий на сайте и построению модели, которая предсказывает, совершит ли пользователь целевое действие.

## Описание проекта

В этом проекте я анализировала данные по визитам на сайт **СберАвтоподписка** и строила модель машинного обучения для предсказания вероятности целевого действия пользователя.

Под целевым действием понимались важные события на сайте, например:
- оставление заявки;
- заказ звонка;
- открытие диалога;
- другие ключевые действия из `ga_hits.event_action`.

## Цель

Цель проекта — построить модель, которая по данным о визите сможет определить, будет ли у пользователя целевое действие на сайте.

Это нужно для:
- оценки эффективности каналов трафика;
- улучшения пользовательского опыта;
- анализа рекламных кампаний;
- повышения конверсии сайта.

## Данные

В проекте использовались два файла (из-за большого размера файла с данными не загружены в репозиторий):

- `ga_sessions.csv` — данные по сессиям;
- `ga_hits.csv` — данные по действиям внутри сессий.

В данных были признаки типа:
- `utm_*` — источники трафика;
- `geo_*` — география;
- `device_*` — устройство;
- `hit_page_path` — страницы сайта;
- `event_action` — действия пользователя.

## Что было сделано

### 1. Загрузка и проверка данных
- загружены оба датасета;
- проверены колонки и типы;
- изучены пропуски и дубликаты;
- найден ключ для объединения данных.

### 2. Создание целевой переменной
- целевая переменная `is_target` была собрана на уровне сессии;
- если в сессии было хотя бы одно целевое действие, то `is_target = 1`;
- иначе `is_target = 0`.

### 3. Очистка и EDA
- обработаны пропуски;
- удалены дубликаты;
- построены базовые графики;
- изучены распределения признаков и корреляции.

### 4. Feature engineering
- добавлены новые признаки;
- подготовлены категориальные и числовые переменные;
- данные приведены к удобному формату для обучения моделей.

### 5. Обучение моделей
Были протестированы две модели:

- baseline: `LogisticRegression`;
- `RandomForestClassifier`.

Результаты:
- **Baseline ROC-AUC: 0.7083**
- **RandomForest ROC-AUC: 0.6357**

В этой задаче baseline-модель оказалась лучше.

## Вывод

Для данной задачи линейная модель показала себя лучше, чем RandomForest.  
Скорее всего, это связано с тем, что после кодирования категориальных признаков данные стали очень разреженными, а логистическая регрессия лучше справилась с такой структурой.
